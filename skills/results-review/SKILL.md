# Results Review Skill

Track post performance after publishing and send structured updates.
Use this when posts are already live and you need recurring outcome reporting.

## Tools Required

- `python3`
- Browser automation: use either
  - `browser` (OpenClaw browser tool), or
  - `browser-tools` CLI (`bin/browser-tools` in agent-skill repo)

Browser-tools command set to use as needed: `start`, `nav`, `pick`, `eval`, `screenshot`, `inspect`, `kill`.

## Cadence (Mandatory)

- Frequency: 3 updates per day
- Duration: 3 consecutive days
- Total updates: 9

Suggested times (local): `09:00`, `14:00`, `20:00`

## Required Files

- Tracker file: `social/YYYY-MM-DD-slug/results-tracking.json`
- Snapshot template: `social/YYYY-MM-DD-slug/results-snapshot.template.json`
- Update outputs: `social/YYYY-MM-DD-slug/results-updates/*.md`
- Latest summary: `social/YYYY-MM-DD-slug/results-latest.md`

## Step 1: Prepare Tracking Artifacts

Run once before the first update:

```bash
python3 skills/results-review/scripts/prepare_results_review.py \
  --social-dir social/YYYY-MM-DD-slug \
  --timezone America/Los_Angeles
```

Then fill `results-tracking.json` `posts` array with every live post URL:
- X article post
- LinkedIn post
- HN post
- Every Reddit post (each subreddit is separate)

Do not skip any destination where the content was published.

## Step 2: Collect Live Metrics

Read post definitions from `results-tracking.json` `posts[]`.
For each post, collect only numeric metrics that the recorder script supports:

- `likes`
- `comments`
- `reposts`
- `bookmarks`
- `views`
- `shares`
- `clicks`
- `upvotes`
- `downvotes`
- `score`

Do not invent new metric keys in snapshots. If you need extra context, use `note`.

### Reddit (Preferred: API/JSON endpoint)
Use the Reddit URL with `.json` suffix:

```text
https://www.reddit.com/r/SUBREDDIT/comments/ID/SLUG/.json
```

Extract:
- `upvotes`: `score`
- `comments`: `num_comments`
- `score`: `score`

If rate-limited, carry forward previous metrics and set `note` (for example `carried forward - Reddit 429`).

### HN (API)
Use Firebase item API:

```text
https://hacker-news.firebaseio.com/v0/item/ITEM_ID.json
```

Extract:
- `score`
- `comments`: `descendants`

### X (Browser - MANDATORY)
X has no stable public metrics API. Use browser automation (`browser` or `browser-tools`).

1. Open the X post URL in browser
2. Read engagement metrics directly from post UI

Map to snapshot keys:
- likes -> `likes`
- replies -> `comments`
- reposts -> `reposts`
- bookmarks -> `bookmarks`
- views -> `views`

### LinkedIn (Browser - MANDATORY)
LinkedIn metrics also require browser access.

1. Open the LinkedIn post URL in browser
2. Read visible engagement metrics

Map to snapshot keys:
- reactions -> `likes`
- comments -> `comments`
- reposts -> `reposts`
- impressions -> `views` (if visible to author)

If browser automation is unavailable for X/LinkedIn:
- carry forward last known metrics when possible
- add `note` such as `browser unavailable`
- continue the run (do not skip writing snapshot/report)

## Step 3: Record Snapshot + Generate Diff Report

Write the collected metrics into a snapshot JSON:

```json
{
  "timestamp": "2026-02-26T09:00:00+01:00",
  "items": [
    {
      "post_id": "x_article",
      "likes": 120,
      "comments": 18,
      "reposts": 7,
      "bookmarks": 12,
      "views": 12400
    },
    {
      "post_id": "linkedin",
      "likes": 42,
      "comments": 3,
      "reposts": 0,
      "views": 22503
    },
    {
      "post_id": "reddit_ai_agents",
      "upvotes": 52,
      "comments": 14,
      "score": 51
    },
    {
      "post_id": "hn",
      "score": 313,
      "comments": 118
    }
  ]
}
```

Then run:

```bash
python3 skills/results-review/scripts/record_results_snapshot.py \
  --social-dir social/YYYY-MM-DD-slug \
  --snapshot social/YYYY-MM-DD-slug/results-snapshot.template.json
```

This appends the snapshot, computes deltas from prior update, and writes:
- timestamped report under `results-updates/`
- `results-latest.md`

Do not manually patch snapshots in `results-tracking.json`.
Always use the script so updates stay schema-compatible (`post_id` or legacy `id`).

If an update failed and you only need to re-sync the latest report from already-stored snapshots:

```bash
python3 skills/results-review/scripts/record_results_snapshot.py \
  --social-dir social/YYYY-MM-DD-slug \
  --refresh-latest-only
```

## Step 4: Send Update to User

Use `results-latest.md` as the source of truth for the user update.
Keep the same per-post table shape generated by `record_results_snapshot.py`:
- `Post`
- `Source`
- `Metrics`
- `Diff from last update`

Do not hand-format a custom report structure that diverges from generated output.

## Checklist

- [ ] Tracker includes every published destination URL
- [ ] Snapshot uses supported metric keys only
- [ ] X/LinkedIn metrics collected via `browser` or `browser-tools` when available
- [ ] Reddit metrics collected via `.json` endpoint (or carried forward with note)
- [ ] HN metrics collected via Firebase API
- [ ] Snapshot recorded via script
- [ ] Diff report shows +/- from previous snapshot
- [ ] Update count remains on cadence (3/day, 3 days)
- [ ] User update sent using generated latest report format
